QuickFeed: AI News Aggregator

A cross-platform mobile app that delivers a personalized, AI-summarized daily briefing instead of an endless feed. I led the technical side for a team of three, owning the stack, schema, and architecture from the start. The build won People's Choice at the 2026 Capstone Fair, voted across all departments by faculty and students.

Problem

Most news apps drown the reader in infinite scroll or hide curation behind a paywall. The goal: deliver the day's signal in a finite, finishable briefing that respects the reader's time. And do it cheaply enough that the unit economics survive growth.

Architecture

Firecrawl → GPT-4o-mini → Supabase → Flutter

A daily Python cron on Railway pulls fresh URLs per topic, scrapes them, summarizes through OpenAI against a strict JSON schema, and writes results to Supabase. The Flutter client only ever reads pre-generated articles. No per-user inference, no per-request LLM calls.

Key Decisions

Database: Chose Supabase over Firebase to keep direct SQL schema control and fine-grained auth. Designed the PostgreSQL schema, indexes, and RLS policies from scratch.
Cost model: A Firecrawl + GPT-4o-mini pipeline that scales with topics, not users. API costs stay flat regardless of how many people install the app.
Backend shape: A scheduled Python job, not a request-response server. The expensive AI work runs once per topic per day; the app just reads from Supabase.
Frontend: Flutter. A single Dart codebase targeting Android and iOS without two separate language stacks.

Engineering Details

SIGALRM hard timeout The cron sets a 10-minute wall-clock alarm so a hung network call can't loop the container forever. Orphaned "running" rows are recovered on the next invocation by a stale-run sweeper.
Strict JSON schema on model output OpenAI responses are validated against a Pydantic schema with enums (pov, sentiment) and exact-count constraints (5 key points). Invalid output triggers one retry on truncated content. If that fails, only the article is dropped, never the whole run.
